# 教师端 - 首页接口文档

## 概述

教师端首页相关接口，提供教师仪表盘、班级管理、任务完成情况、学生学习趋势等数据。

**基础路径**: `/api/teacher-home`

**认证方式**: 除特别说明外，所有接口需在 Header 中携带 `Authorization: Bearer <token>`

---

## 接口列表

| 序号 | 接口 | 方法 | 说明 |
|:---:|------|:----:|------|
| 1 | `/dashboard` | GET | 获取教师仪表盘数据（顶部概览） |
| 2 | `/level-distribution` | GET | 获取班级等级分布 |
| 3 | `/task-completion` | GET | 获取各班级任务完成率对比 |
| 4 | `/activity-trend` | GET | 获取学生活跃度趋势（最近7天） |
| 5 | `/error-type-distribution` | GET | 获取错题类型分布 |

---

## 接口详情

### 1. 获取教师仪表盘数据（顶部概览）

**接口地址**: `GET /api/teacher-home/dashboard`

**请求头**:
```
Authorization: Bearer <token>
```

**响应示例**:
```json
{
  "code": 200,
  "data": {
    "totalStudents": 45,
    "totalClasses": 2,
    "pendingApplications": 0,
    "pendingTasks": 0
  }
}
```

**字段说明**:

| 字段 | 类型 | 说明 |
|------|:----:|------|
| totalStudents | Number | 所带班级学生总数 |
| totalClasses | Number | 所带班级数量 |
| pendingApplications | Number | 待审批申请数 |
| pendingTasks | Number | 待处理任务数 |

---

### 2. 获取班级等级分布

**接口地址**: `GET /api/teacher-home/level-distribution`

**请求头**:
```
Authorization: Bearer <token>
```

**响应示例**:
```json
{
  "code": 200,
  "data": [
    { "classLevel": "A", "classCount": 1 },
    { "classLevel": "B", "classCount": 1 },
    { "classLevel": "C", "classCount": 1 },
    { "classLevel": "D", "classCount": 1 }
  ]
}
```

**字段说明**:

| 字段 | 类型 | 说明 |
|------|:----:|------|
| classLevel | String | 班级等级 (A/B/C/D) |
| classCount | Number | 该等级班级数量 |

---

### 3. 获取各班级任务完成率对比

**接口地址**: `GET /api/teacher-home/task-completion`

**请求头**:
```
Authorization: Bearer <token>
```

**响应示例**:
```json
{
  "code": 200,
  "data": {
    "classNames": ["A级-英语精英班", "B级-英语进阶班", "C级-英语提高班", "D级-英语基础班"],
    "completionRates": [85, 78, 92, 65]
  }
}
```

**字段说明**:

| 字段 | 类型 | 说明 |
|------|:----:|------|
| classNames | Array | 班级名称列表 |
| completionRates | Array | 各班级任务完成率(%) |

---

### 4. 获取学生活跃度趋势（最近7天）

**接口地址**: `GET /api/teacher-home/activity-trend`

**请求头**:
```
Authorization: Bearer <token>
```

**响应示例**:
```json
{
  "code": 200,
  "data": {
    "days": ["周一", "周二", "周三", "周四", "周五", "周六", "周日"],
    "completedTasks": [220, 182, 191, 234, 290, 330, 310],
    "exercises": [150, 232, 201, 154, 190, 330, 410]
  }
}
```

**字段说明**:

| 字段 | 类型 | 说明 |
|------|:----:|------|
| days | Array | 日期标签（周一至周日） |
| completedTasks | Array | 每日完成任务数 |
| exercises | Array | 每日练习次数 |

---

### 5. 获取错题类型分布

**接口地址**: `GET /api/teacher-home/error-type-distribution`

**请求头**:
```
Authorization: Bearer <token>
```

**响应示例**:
```json
{
  "code": 200,
  "data": [
    { "type": "选择题", "count": 335 },
    { "type": "填空题", "count": 234 },
    { "type": "单词拼写", "count": 148 }
  ]
}
```

**字段说明**:

| 字段 | 类型 | 说明 |
|------|:----:|------|
| type | String | 错题类型 |
| count | Number | 该类型错题数量 |

---

## 响应码说明

| code | 说明 |
|:----:|------|
| 200 | 请求成功 |
| 401 | 未登录或 token 无效 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

---

## 模拟用户数据

接口会根据请求头中 token 解析出的用户名返回对应教师的数据：

| 用户名 | 说明 |
|--------|------|
| teacher | 默认教师用户 |
| teacherLi | 李老师 |
| teacherWang | 王老师 |

> 登录接口: `POST /api/auth/login`，请求体 `{ "username": "teacher", "password": "teacher" }`
