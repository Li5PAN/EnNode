# 教师端 - 我的班级接口文档

## 概述

教师班级管理相关接口，提供班级概览、班级列表、创建班级、发布任务等功能。

**基础路径**: `/api/teacher-class`

**认证方式**: 除特别说明外，所有接口需在 Header 中携带 `Authorization: Bearer <token>`

---

## 接口列表

| 序号 | 接口 | 方法 | 说明 |
|:---:|------|:----:|------|
| 1 | `/overview` | GET | 获取班级概览数据 |
| 2 | `/list` | GET | 获取班级列表（当前老师） |
| 3 | `/all` | GET | 获取所有班级列表（管理员用） |
| 4 | `/create` | POST | 创建班级（需审核） |
| 5 | `/pending` | GET | 获取待审核班级列表（管理员） |
| 6 | `/approve/:classId` | POST | 审核通过班级（管理员） |
| 7 | `/reject/:classId` | POST | 拒绝班级（管理员） |
| 8 | `/delete/:classId` | DELETE | 删除班级 |
| 9 | `/publish-task` | POST | 发布任务 |

---

## 接口详情

### 1. 获取班级概览数据

**接口地址**: `GET /api/teacher-class/overview`

**请求头**:
```
Authorization: Bearer <token>
```

**响应示例**:
```json
{
  "code": 200,
  "data": {
    "totalClasses": 2,
    "avgCompletionRate": 80,
    "pendingApplications": 5
  }
}
```

**字段说明**:

| 字段 | 类型 | 说明 |
|------|:----:|------|
| totalClasses | Number | 班级数量 |
| avgCompletionRate | Number | 平均完成率(%) |
| pendingApplications | Number | 待审批入班申请数 |

---

### 2. 获取班级列表（当前老师）

**接口地址**: `GET /api/teacher-class/list`

**请求头**:
```
Authorization: Bearer <token>
```

**请求参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|:----:|:----:|------|
| classLevel | String | 否 | 班级等级 (A/B/C/D) |
| pageNum | Number | 否 | 页码，默认1 |
| pageSize | Number | 否 | 每页数量，默认100 |

**响应示例**:
```json
{
  "code": 200,
  "rows": [
    {
      "classId": 1,
      "className": "A级-英语精英班",
      "classLevel": "A",
      "currentStudents": 25,
      "maxStudents": 30,
      "taskCount": 15,
      "createTime": "2026-03-01",
      "pendingApplicationCount": 2,
      "teacherId": "teacher",
      "teacherName": "张老师"
    }
  ],
  "total": 2
}
```

**字段说明**:

| 字段 | 类型 | 说明 |
|------|:----:|------|
| classId | Number | 班级ID |
| className | String | 班级名称 |
| classLevel | String | 班级等级 (A/B/C/D) |
| currentStudents | Number | 当前学生数 |
| maxStudents | Number | 最大学生数 |
| taskCount | Number | 任务数量 |
| createTime | String | 创建时间 (YYYY-MM-DD) |
| pendingApplicationCount | Number | 待审批申请数 |
| teacherId | String | 老师ID |
| teacherName | String | 老师姓名 |

### 4. 创建班级（需管理员审核）

**接口地址**: `POST /api/teacher-class/create`

**请求头**:
```
Authorization: Bearer <token>
```

**请求体**:
```json
{
  "className": "A级-英语精英班",
  "classLevel": "A",
  "maxStudents": 30,
  "taskRequirement": 10
}
```

**字段说明**:

| 字段 | 类型 | 必填 | 说明 |
|------|:----:|:----:|------|
| className | String | 是 | 班级名称 |
| classLevel | String | 是 | 班级等级 (A/B/C/D) |
| maxStudents | Number | 是 | 最大学生数 |
| taskRequirement | Number | 否 | 任务要求数 |

**响应示例**:
```json
{
  "code": 200,
  "msg": "班级创建成功，等待管理员审核",
  "data": {
    "classId": 1713789000000,
    "status": "pending"
  }
}
```

> 班级创建后状态为 `pending`，需管理员审核通过后才会在班级列表中显示。

### 6. 发布任务

**接口地址**: `POST /api/teacher-class/publish-task`

**请求头**:
```
Authorization: Bearer <token>
```

**请求体**:
```json
{
  "classId": 1,
  "taskName": "第三单元单词测试",
  "taskType": "2",
  "startTime": "2026-04-21 09:00:00",
  "deadline": "2026-04-28 23:59:59",
  "questions": [
    {
      "type": "choice",
      "content": "What is the meaning of 'apple'?",
      "options": ["苹果", "香蕉", "橙子", "葡萄"],
      "correctIndexes": [0]
    },
    {
      "type": "fillBlank",
      "content": "The capital of China is ______.",
      "answer": "Beijing"
    },
    {
      "type": "spelling",
      "content": "请拼写单词：学校",
      "answer": "school"
    }
  ]
}
```

**字段说明**:

| 字段 | 类型 | 必填 | 说明 |
|------|:----:|:----:|------|
| classId | Number | 是 | 班级ID |
| taskName | String | 是 | 任务名称 |
| taskType | String | 是 | 任务类型 (1:单词学习, 2:单词测试, 3:综合练习) |
| startTime | String | 是 | 开始时间 (YYYY-MM-DD HH:mm:ss) |
| deadline | String | 是 | 截止时间 (YYYY-MM-DD HH:mm:ss) |
| questions | Array | 是 | 题目列表 |

**questions 数组字段说明**:

| 字段 | 类型 | 说明 |
|------|:----:|------|
| type | String | 题型 (choice/fillBlank/spelling) |
| content | String | 题目内容 |
| options | Array | 选项列表（选择题用） |
| correctIndexes | Array | 正确答案索引（选择题用） |
| answer | String | 正确答案（填空/拼写题用） |

**响应示例**:
```json
{
  "code": 200,
  "msg": "任务发布成功",
  "data": {
    "taskId": 1713789000000
  }
}
```

---

## 响应码说明

| code | 说明 |
|:----:|------|
| 200 | 请求成功 |
| 400 | 请求参数错误 |
| 401 | 未登录或 token 无效 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

## 模拟用户数据

接口会根据请求头中 token 解析出的用户名返回对应教师的数据：

| 用户名 | 说明 |
|--------|------|
| teacher | 张老师 |
| teacherLi | 李老师 |
| teacherWang | 王老师 |

> 登录接口: `POST /api/auth/login`，请求体 `{ "username": "teacher", "password": "teacher" }`
